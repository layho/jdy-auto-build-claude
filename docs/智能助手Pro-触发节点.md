# 智能助手Pro - 触发节点

> 来源: https://hc.jiandaoyun.com/zh_cn/doc/18478

## 概述

触发节点是「智能助手Pro」中的核心触发器，负责控制整个工作流的启动与停止。

## 五种触发方式

### 1. 表单触发
当表单中的数据发生新增、修改、删除等变化，或流程状态改变且满足条件时，自动触发。

支持的触发动作：
- **新增数据时** — 在触发表单中填报数据
- **修改数据时** — 修改触发表单中的已有数据
- **删除数据时** — 删除触发表单中的某条数据
- **流程结束时** — 流程流转完成（含手动结束）
- **节点流转时** — 在流程节点"提交"或"回退"时触发

注意：若设置了触发条件，系统会先校验条件是否满足，通过后才触发执行。

### 2. 定时触发
从设定时间或表单中指定时间开始，按固定频率周期性触发。

两种触发时间类型：
- **自定义时间** — 从管理员设定的时间开始触发
- **根据表单日期时间字段** — 从表单数据中的日期时间开始触发

### 3. HTTP触发
当指定URL接收到来自外部系统的数据推送时触发。URL由智能助手Pro生成并对外提供。

### 4. 按钮触发
用户点击自定义按钮后触发执行后续节点操作。

## 关键概念

| 概念 | 说明 |
|------|------|
| 触发条件 | 在触发动作基础上附加的校验规则，条件不满足则不触发 |
| 触发方式 | 表单、定时、HTTP、按钮四种入口 |
| 执行节点 | 触发后向下执行的任务节点（新增/修改/删除数据、条件分支等） |
