# 智能助手Pro - 功能入口与设计步骤

> 来源: https://hc.jiandaoyun.com/zh_cn/doc/19049

## 功能入口

### 从表单进入
路径：**表单编辑 → 扩展功能 → 智能助手**
- 未设置过：点击「新建智能助手」
- 已设置过：可持续发类型/触发&执行动作筛选，支持编辑、启用/停用、查看日志、复制、删除

### 从应用后台进入
路径：**工作台 → 应用 → 应用后台** → 高级功能中的「智能助手」
- 支持分类/名称/表单/启用状态筛选

## 核心设计流程

智能助手Pro通过一个**触发节点**启动，再通过多个**执行节点**执行任务序列。

### 1. 设置触发节点

| 触发方式 | 说明 |
|---|---|
| **表单触发** | 表单数据或流程状态变化时触发 |
| **定时触发** | 按设定时间或表单日期字段周期触发 |
| **HTTP触发** | 外部系统推送数据到指定URL时触发 |
| **按钮触发** | 点击自定义按钮时触发 |

### 2. 添加并设置执行节点

通过点击「+」按钮顺序添加：

| 节点类型 | 作用 |
|---|---|
| 新增数据 | 向目标表单添加数据 |
| 修改数据 | 修改已有数据，找不到时可新增 |
| 删除数据 | 删除满足条件的数据 |
| 查询单条/多条数据 | 查询结果可被后续节点调用 |
| 条件分支 | 按不同条件执行不同路径 |
| 循环容器 | 对多组数据重复执行操作 |
| 计算节点 | 对前置数据进行公式计算 |
| 结束流程 | 自动终止流程数据处理 |
| 插件节点 | 调用外部插件 |
| 生成打印文件 | 对单条数据打印 |
| 发送消息 | 向指定成员发送消息 |

### 3. 保存并启用

完成设计后，点击「保存并启用」使智能助手Pro运行。

## 重要注意事项

1. **互不触发**：智能助手Pro与旧版智能助手之间不会互相触发
2. **流程与数据拆分**：流程表单中，修改数据和变更流程可分别触发
3. **执行失败处理**：需在执行日志中查看失败日志并手动调整
